# 家长端时间管理设计

日期：2026-06-23

## 背景

家长端当前有“上课记录”页面，按列表展示已经产生的课堂记录。新的目标是把它替换为时间管理页面：家长可以像老师端排课页一样用月视图和周视图查看课程安排，并在右侧查看课程详情、老师评语和课堂内容。

同时，家长需要能添加自己的其他课程安排，例如校外英语课或训练课。这些安排只服务家长自己查看，不影响老师端排课、考勤、课时统计和学习进度。

## 目标

- 用新的“时间管理”页面替换家长端“上课记录”入口。
- 支持月视图和周视图，视觉和老师端排课页保持一致。
- 展示未来老师已排课程和历史课堂记录。
- 老师已排课程对家长只读，不支持家长修改。
- 点击老师课程后，在右侧展示详细时间、老师、科目、课程内容、老师评语、标签和薄弱点。
- 家长可以新增、编辑和删除自己的个人课程安排。
- 家长个人安排只对创建它的家长可见，不回流老师端。

## 非目标

- 不允许家长修改老师端排课。
- 不允许家长进行考勤操作。
- 家长个人安排不写入考勤记录。
- 家长个人安排不扣课时。
- 家长个人安排不生成课堂反馈、薄弱点或复习计划。
- 本次不做复杂冲突检测，只允许同一天同一时间存在多条安排。

## 页面入口

沿用现有路径 `/parent/lessons`，但菜单文案从“上课记录”改为“时间管理”。

这样可以替换原有家长上课记录体验，同时避免迁移旧链接。原列表式上课记录不再作为独立页面保留，历史课堂信息通过日历详情查看。

## 页面结构

页面主体采用老师端排课页的日历布局：

- 顶部工具栏：今天、上一页、下一页、当前月份或周范围、月/周切换。
- 左侧主区域：月视图或周视图。
- 右侧详情区：展示所选日期或所选事项详情。

日历事项分两类：

- 老师课程：来自现有 `Schedule`、`Attendance` 和学习关系数据。
- 个人安排：来自家长自己创建的个人日程数据。

点击日期但未选中事项时，右侧展示当天全部事项列表。点击具体事项时，右侧展示该事项详情。

## 数据设计

新增 `ParentScheduleItem` 表，用于保存家长个人安排。

建议字段：

- `id`
- `workspaceId`
- `parentId`
- `studentId`
- `learningLinkId`，可选
- `title`
- `date`
- `startTime`
- `endTime`
- `notes`
- `createdAt`
- `updatedAt`

字段含义：

- `parentId` 决定个人安排的所有者。
- `studentId` 决定这条安排属于哪个孩子。
- `learningLinkId` 可选，用于在存在多个学习关系时和孩子、老师、科目筛选保持一致。
- `title` 用于显示家长自建安排名称，例如“英语课”“篮球训练”。
- `date`、`startTime`、`endTime` 用于日历定位和右侧详情展示。

老师课程继续使用现有数据：

- `Schedule` 表示老师排课。
- `Attendance` 表示当天出勤状态和课堂反馈。
- `LearningLink` 决定家长是否有权查看对应孩子、老师、科目和课程。

## 接口设计

新增 `/api/parent/schedule-items`。

### GET

返回当前家长可见的日历事项，包含两类统一结构：

- 老师课程事项。
- 家长个人安排事项。

老师课程读取规则：

- 只读取当前家长启用学习关系下的学生。
- 固定排课按 `dayOfWeek` 展示到日历中。
- 灵活排课按 `date` 展示到日历中。
- 关联当天同一学生、同一排课的 `Attendance`，用于展示状态、课程内容和老师评语。

个人安排读取规则：

- 只读取 `parentId` 等于当前家长的记录。
- 只读取当前工作区记录。

### POST

创建家长个人安排。请求字段：

- `studentId`
- `learningLinkId`，可选
- `title`
- `date`
- `startTime`
- `endTime`
- `notes`

服务端校验：

- 当前用户必须是家长。
- `studentId` 必须属于当前家长可访问的孩子。
- 如果传入 `learningLinkId`，必须属于当前家长启用学习关系。

### PATCH

编辑家长个人安排。只能编辑当前家长自己创建的记录。

允许更新：

- `studentId`
- `learningLinkId`
- `title`
- `date`
- `startTime`
- `endTime`
- `notes`

### DELETE

删除家长个人安排。只能删除当前家长自己创建的记录。

## 权限规则

- 家长只能查看自己启用学习关系下的老师课程。
- 家长不能通过页面或接口修改老师课程。
- 家长不能通过页面或接口写入老师课程考勤。
- 家长只能查看、创建、编辑和删除自己的 `ParentScheduleItem`。
- 老师端 `/schedule` 不读取 `ParentScheduleItem`。
- 管理员端暂不提供家长个人安排管理入口。

## 右侧详情

### 老师课程

展示内容：

- 课程名称或学生姓名。
- 孩子姓名。
- 老师姓名。
- 科目。
- 日期。
- 开始时间和结束时间。
- 课程类型：固定课程或临时课程。
- 考勤状态：待上课、出勤、请假或补课。
- 上课内容。
- 老师评语。
- 内容标签。
- 反馈标签。
- 薄弱点标签。

展示规则：

- 未来未产生考勤的课程显示“待上课”。
- 出勤和补课记录显示课堂内容、老师评语和标签。
- 请假记录显示请假状态，课堂内容和老师评语允许为空。
- 所有老师课程详情只读，不显示编辑、删除或考勤按钮。

### 个人安排

展示内容：

- 标题。
- 孩子姓名。
- 日期。
- 开始时间和结束时间。
- 备注。
- 类型标识：个人安排。

个人安排详情区提供编辑和删除按钮。

## 视觉区分

日历里使用不同样式区分事项来源：

- 老师课程：蓝色，标注“老师安排”。
- 个人安排：绿色或灰绿色，标注“个人安排”。

月视图中每一天展示有限条事项，超出时显示剩余数量。周视图按小时展示事项，和老师端保持一致。

## 空状态

- 当前日期没有事项时，右侧显示“暂无课程安排”。
- 老师课程未上课时，反馈区显示“待老师课后填写”。
- 家长还没有可访问学习关系时，页面显示“暂无可查看的学习关系”。
- 家长还没有孩子时，新增个人安排入口不可用，并提示先联系老师或管理员绑定孩子。

- 检查家长侧边栏把“上课记录”替换为“时间管理”。

- 检查家长可以看到未来老师固定排课。
- 检查家长可以看到老师临时排课。
- 检查老师课程详情只读，没有编辑、删除和考勤按钮。
- 检查已上课课程展示课堂内容、老师评语、标签和薄弱点。
- 检查未上课课程显示“待上课”。
- 检查家长可以新增、编辑和删除个人安排。
- 检查家长个人安排只在当前家长账号可见。
- 检查家长不能通过接口编辑或删除其他家长的个人安排。
- 检查老师端 `/schedule` 不显示家长个人安排。
- 运行构建检查页面和接口类型。

## 实施备注

实现时可以先保留老师端 `src/app/schedule/page.tsx` 的布局思路，但建议将家长端日历做成独立只读组件，避免把老师端新增排课、考勤弹窗、标签管理等操作误带到家长端。

家长端需要的是“查看正式课程 + 管理个人安排”，不是老师端排课考勤能力的子集。保持这个边界可以降低后续权限和数据污染风险。
