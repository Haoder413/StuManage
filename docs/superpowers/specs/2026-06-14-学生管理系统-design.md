# 数学补习班学生管理系统 — 设计文档

## 1. 项目概述

单人教师的小型数学补习班管理系统，支持小班定制化教学场景。核心需求是**成绩管理**和**学习进度追踪**，配合排课考勤、报表导出、沟通记录等辅助模块。

**适用场景**

- 一位老师管理 10-50 个学生
- 支持两种教学模式：寒暑假上新课（按知识点大纲推进）、平时补薄弱（针对性解决 + 艾宾浩斯复习提醒）
- 课程采用"基础课程框架 + 学生个性化调整"的混合模式
- 未来可扩展为 Web 端 + 小程序端双入口

**技术选型**

| 层 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js (App Router) | 全栈合一，一人开发效率最高；API Routes 预留小程序接口 |
| 数据库 | SQLite (via Prisma) | 零配置、零运维，单人场景完全够用 |
| UI | Tailwind CSS + Shadcn/ui | 快速出精致界面 |
| ORM | Prisma | 类型安全、迁移方便，未来可无缝切换到 PostgreSQL |
| 部署 | Vercel | 一行命令部署，支持 Serverless |

## 2. 整体架构

```
+---------------------------------------+
|         Next.js App Router            |
|  +-------------+  +----------------+  |
|  |   页面层     |  |   API 层       |  |
|  |  app/       |  |  app/api/      |  |
|  |  Server     |  |  RESTful       |  |
|  |  Components |  |  接口          |  |
|  +-------------+  +----------------+  |
|           |          |                |
|  +--------------------------------+   |
|  |      Prisma ORM               |   |
|  |      lib/prisma.ts            |   |
|  +--------------------------------+   |
|           |                           |
|  +--------------------------------+   |
|  |         SQLite (.db)           |   |
|  +--------------------------------+   |
+---------------------------------------+
```

**关键设计原则**

- **页面层**使用 React Server Components 减少客户端 JavaScript，交互密集区域（表单、图表）用 Client Components
- **API 层**为 RESTful 风格，作为面向未来的小程序统一接口层
- **数据访问**全部通过 Prisma，保证类型安全和迁移能力

## 3. 数据模型

共 11 张表，覆盖所有业务模块：

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| students | 学生档案 | 姓名, 年级, 家长联系方式, 入学日期, 上课频次, 学费, 备注 |
| courses | 课程定义 | 名称, 描述, 类型 (固定/定制) |
| student_courses | 学生选课关联 | 学生, 课程, 开始日期, 状态, 定制内容 (JSON) |
| exams | 考试成绩 | 学生, 考试名称, 类型 (摸底/阶段/随堂), 得分, 满分, 日期 |
| knowledge_points | 知识点大纲 (树形) | 课程, 名称, 父级 id, 排序序号 |
| student_kp_progress | 知识点进度 | 学生, 知识点, 状态 (未学/学习中/已掌握), 完成时间 |
| weak_points | 薄弱点 | 学生, 关联知识点 (可选), 描述, 状态 (待解决/已掌握) |
| review_schedules | 复习计划 (艾宾浩斯) | 薄弱点, 复习阶段 (1-6), 上次复习时间, 下次复习时间, 状态 |
| schedules | 排课 | 学生, 类型 (固定/灵活), 时间段, 有效期 |
| attendance | 考勤记录 | 学生, 排课, 日期, 状态 (出勤/请假/补课) |
| communication_logs | 沟通记录 | 学生, 日期, 方式 (电话/微信), 内容 |

**知识点树形结构示例**

```
七年级上册
+-- 有理数
|   +-- 正负数概念
|   +-- 数轴与绝对值
|   +-- 有理数运算
+-- 整式
|   +-- 单项式与多项式
|   +-- 合并同类项
+-- 一元一次方程
    +-- 方程概念
    +-- 解方程与应用题
```

**艾宾浩斯复习阶段**

| 阶段 | 间隔 | 说明 |
|------|------|------|
| 第1次 | 1 天后 | 首次巩固 |
| 第2次 | 3 天后 | 短期记忆 |
| 第3次 | 7 天后 | 一周回顾 |
| 第4次 | 15 天后 | 中期巩固 |
| 第5次 | 30 天后 | 月度复习 |
| 第6次 | 60 天后 | 长期强化 |

每次复习确认"记住了"进入下一阶段；发现"忘了"则重置到"待解决"状态。

## 4. 页面结构与导航

### 侧边栏导航

8 个入口，按使用频率排序：

1. **仪表盘** — 今日课表、待办复习提醒、关键数据概览
2. **学生管理** — 学生列表、搜索筛选、增删改、详情查看
3. **课程管理** — 课程定义、知识点树编辑
4. **成绩管理** — 成绩录入、成绩单查看、趋势图表
5. **学习进度** — 寒暑假知识点跟踪 + 平时薄弱点清单 + 复习日历
6. **排课考勤** — 固定排课 / 灵活排课 + 出勤记录
7. **报表导出** — 按学生/时段筛选，导出成绩单或进度报告
8. **沟通记录** — 按学生查看和添加沟通记录

### 统一布局

- 深色侧边栏 (slate-900) + 浅灰主内容区 (slate-50)
- 统计卡片四列网格布局
- 待办事项和课程安排双列卡片布局
- 响应式设计，移动端侧边栏可折叠

## 5. 核心功能详情

### 5.1 学生管理

- **列表页**: 表格展示所有学生，支持按姓名/年级/课程搜索筛选，每行包含快速操作（编辑/查看详情）
- **详情页**: 学生完整档案 + 关联数据概览（成绩、进度、出勤、沟通记录汇总）
- **新增/编辑**: 表单包含姓名、年级、家长联系方式、入学日期、上课频次、学费信息、备注

### 5.2 课程管理

- **课程列表**: 固定课程与定制课程分别展示
- **知识点编辑器**: 树形结构，拖拽排序，嵌套层级不限
- **知识点大纲**: 每个课程拥有独立的知识点树，作为寒暑假教学大纲

### 5.3 成绩管理

- **录入**: 选择学生、考试类型（摸底/阶段/随堂）、输入得分和满分。支持批量录入（一次考试选多个学生）
- **成绩单**: 按学生查看所有考试成绩列表，带趋势折线图
- **考试管理**: 按考试维度查看，显示参与学生和分数排名分布

### 5.4 学习进度

**寒暑假模式（上新课）**

- 展示课程知识点树，每个节点显示学生当前状态（未学/学习中/已掌握）
- 一键切换状态，支持批量标记
- 进度百分比概览：已掌握 / 进行中 / 未开始

**平时模式（补薄弱）**

- 每个学生独立的薄弱点清单
- 添加薄弱点：关联知识点（可选）+ 文字描述
- 标记"已掌握"后自动创建艾宾浩斯复习计划
- 复习提醒：仪表盘展示待办提醒，逾期标红

### 5.5 排课考勤

- **固定排课**: 设置每周固定时间段，系统自动生成上课日历
- **灵活排课**: 手动添加单次课程安排
- **考勤记录**: 每次课程记录出勤状态（出勤/请假/补课）

### 5.6 报表导出

- 成绩单：选择学生 + 时间段，导出含考试列表和趋势的 PDF/Excel
- 进度报告：选择学生，导出知识点掌握情况和薄弱点报告

### 5.7 沟通记录

- 按学生查看历史沟通记录时间线
- 添加记录：选择方式（电话/微信/面谈）+ 填写内容

## 6. 核心工作流

### 成绩录入流程

1. 老师进入"成绩管理"页面
2. 选择"新建考试"
3. 填写考试名称、选择类型（摸底/阶段/随堂）、日期
4. 选择参与学生（多选）
5. 输入每个学生的得分（或批量复制满分/默认分）
6. 提交，自动关联到学生成绩记录

### 薄弱点复习提醒流程

1. 老师在学习进度页面添加学生的薄弱点
2. 经过教学确认薄弱点已掌握 -> 标记"已掌握"
3. 系统自动创建艾宾浩斯复习计划（第1阶段）
4. 按计划时间到达时，仪表盘展示复习提醒
5. 老师点击"已完成复习"并确认学生掌握情况
6. 掌握 -> 进入下一阶段；未掌握 -> 重置为待解决
7. 直到第6阶段通过 -> 薄弱点标记为"长期已掌握"

## 7. 错误处理与边界情况

- **成绩录入**: 得分不应超过满分；考试日期不应在未来太远（防手误）
- **知识点调整**: 课程知识点树变更后，已有进度记录应平滑迁移（保留原知识点 id）
- **排课冲突**: 同一时间段同一老师不应有两个课程，前端做校验提示
- **数据删除**: 学生删除需二次确认；关联数据做软删除或提示关联影响
- **空状态**: 各列表页展示友好的空状态提示和引导操作

## 8. 未来扩展

- **小程序端**: API Routes 已预留，小程序直接调用 /api/* 接口
- **数据库升级**: Prisma 从 SQLite 切换到 PostgreSQL 只需改 DATABASE_URL 连接字符串
- **多教师支持**: 添加教师表和权限校验，适配多教师场景
- **消息推送**: 在沟通记录基础上，未来可接入短信/微信模板消息推送
