# 统一薄弱点复习设计

## 目标

把“薄弱点 / 复习计划 / 历史薄弱点”合并成一个“薄弱点复习”入口，让老师不用理解三个分类，只看一张列表并直接处理。

## 设计

- 学习进度详情页保留“知识点进度”和“薄弱点复习”两个主 tab。
- “薄弱点复习”内显示当前薄弱点和历史薄弱点。
- 顶部用筛选切换：全部、待复习、当前薄弱、巩固中、已完成。
- 每条薄弱点显示描述、状态、下次复习、逾期状态、已复习次数、最近复习时间。
- 每条记录右侧直接显示可用操作：
  - 当前薄弱点：记住了、忘了、已掌握。
  - 巩固中：记住了、忘了。
  - 已完成：只展示历史。

## 范围

不改数据库结构，不改后端接口语义，只重组前端流程。
