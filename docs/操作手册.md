# 学生管理系统操作手册

更新时间：2026-06-30

本手册面向日常使用者，主要说明教师端、管理员端、家长端和微信小程序怎么使用。项目功能更新后，建议同步更新本文，避免代码已经变化、操作说明仍停留在旧版本。

## 1. 系统入口与账号

### 1.1 访问入口

- 本地开发环境通常访问：`http://localhost:3000`
- 服务器线上入口：`https://taotaomath.top` 或 `https://www.taotaomath.top`
- 微信小程序名称：`淘淘数学`

未登录时系统会自动跳转到登录页。

### 1.2 账号角色

系统目前有四类角色：

| 角色 | 可用范围 |
| --- | --- |
| 管理员 | 可使用教师端全部功能，并额外管理账号、学习关系、资料权限和数据库备份。 |
| 老师 | 使用学生、课程、成绩、学习进度、排课考勤、资料、沟通记录等教学管理功能。 |
| 家长 | 进入家长端，只能查看被授权孩子的数据，也可以提交成绩和维护自己的时间安排。 |
| 演示账号 | 用于展示系统功能，数据与真实工作区隔离。 |

如果执行过初始化演示数据，默认账号通常为：

| 角色 | 账号 | 密码 |
| --- | --- | --- |
| 管理员 | `admin` | `admin123` |
| 老师 | `teacher` | `teacher123` |
| 家长 | `parent` | `parent123` |
| 演示 | `demo` | `demo123` |

正式使用前，请先在“系统设置/账号设置”里修改默认密码。

### 1.3 工作区隔离

真实数据、演示数据通过工作区隔离。管理员创建账号、资料和学习关系时要注意工作区选择：

- 真实学生、真实课程、真实家长应放在真实工作区。
- 演示账号和演示数据应放在演示工作区。
- 家长只能看到自己所在工作区内、被授权或被学习关系绑定的学生数据。

## 2. 推荐日常录入顺序

第一次整理数据时，建议按这个顺序操作：

1. 先创建课程，设置课程类型、课程时间和知识点大纲。
2. 再创建学生；学生建档时不选择课程。
3. 通过课程管理、账号与学习关系或学生详情维护学生与课程的归属关系。
4. 如果需要家长端，管理员创建家长账号，并建立“家长-学生-老师-科目/课程”的学习关系。
5. 后续上课当天在“排课考勤”中记录出勤、请假、补课、课堂内容、课堂反馈和薄弱点。
6. 在“学习进度”中维护知识点掌握状态和薄弱点复习。
7. 在“成绩管理”中维护或审核成绩。
8. 在“资料中心”上传资料，并给课程授权。

## 2.1 课堂视频回放

课堂视频回放嵌入现有课堂流程，不作为第一版独立页面。

- 老师端入口在“排课考勤”的课后反馈弹窗中，一次出勤或补课记录可以关联一个课堂录制视频。
- 家长端入口在“时间管理”的老师课程详情中，家长点击具体课程后观看该节课回放。
- 权限沿用学习关系，家长只能观看自己启用学习关系下孩子对应课堂的视频。
- 当前支持上传 `mp4`、`webm`、`mov`、`m4v` 视频；网页端以在线播放为主，不提供家长下载入口。
- 课堂视频存储支持通过服务器环境变量切换：默认本地存储，也可以切到腾讯云 VOD；后续预留 COS + CDN 迁移。
- 老师保存带视频的课后反馈时，弹窗会显示视频上传进度；上传期间不要关闭页面，等待进度完成后系统会自动保存并关闭弹窗。
- 如果上传时提示服务器上传上限不足，通常是线上 Nginx 上传大小限制，需要按部署文档调整后再试；如果提示视频超过系统上传上限，则需要压缩视频，或由管理员调整服务器 `LESSON_VIDEO_MAX_MB` 配置。
- 如果后续增加“从链接导入”入口，老师可先把视频放到 NAS 或其他可公网访问地址，再在课后反馈中填入视频 URL，由系统调用 VOD 拉取上传并绑定到本节课；当前正式入口仍以网站上传为准。
- 腾讯会议普通回放播放页链接不等同于视频文件直链，通常不能直接作为 VOD 拉取源；如果使用腾讯会议录制，应优先获取可下载的录制文件直链，或先下载到 NAS/对象存储后再通过“从链接导入”绑定课堂。
- 当前不做独立“回放库”，后续视频数量多时再增加按孩子、课程和日期筛选的独立入口。

## 2.2 公开首页与登录入口

- 网站根路径 `/` 为公开首页，不再直接跳转登录页。
- 公开首页顶部展示网站免责声明，下方直接展示公开资料中心内容，不再显示“资料中心”按钮。
- 公开资料中心读取服务器独立目录 `storage/public-materials`，线上共享目录为 `/opt/student-management/shared/storage/public-materials`；管理员可以直接把公开展示文件上传到该目录。
- 公开资料中心支持展示 `.pdf`、`.html`、`.htm`、`.doc`、`.docx`、`.png`、`.jpg`、`.jpeg`、`.webp` 文件；HTML/HTM 按动画展示，PDF/DOC/DOCX 按习题展示，图片按资料展示。
- 登录页不在公开首页展示入口，旧地址 `/login` 不再显示登录表单；需要直接输入隐藏登录地址 `/teacher-login-2026` 才能访问。
- 登录成功后，老师、管理员和演示账号进入后台仪表盘 `/dashboard`；家长账号进入家长端。
- 公开首页底部悬挂 ICP 备案号 `辽ICP备2026013992号`，链接到工信部备案查询网站 `https://beian.miit.gov.cn/`。
- 当前公开口径：本网站为个人学习分享博客，本站只面向老师提供服务，仅可查看站长发布内容，全站对外公开展示的知识点动画、自编习题、学习笔记均由网站备案持有人独立创作发布；本站动画，课件均由 AI 辅助生成，本站未开放用户注册，用户无法上传内容，无法发布、分享，网站无公开栏目，无用户公开发布、评论互动功能。

## 3. 教师端/管理员端功能

### 3.1 仪表盘

入口：左侧导航“仪表盘”，后台地址为 `/dashboard`

仪表盘用于快速查看当天整体情况：

- 学生总数
- 考试记录数量
- 待复习提醒
- 今日课程
- 逾期或到期的薄弱点复习

如果“待复习提醒”出现记录，建议进入“学习进度”对应学生详情处理。

### 3.2 学生管理

入口：左侧导航“学生管理”

学生管理首页以卡片形式展示学生。卡片显示：

- 姓名、年级
- 所属课程
- 上课频次
- 入学日期
- 家长联系方式

#### 新增学生

点击“新增学生”，填写：

- 姓名
- 年级
- 家长联系方式
- 入学日期
- 上课频次
- 学费
- 备注

新增学生时不再选择课程。学生建档后，课程归属通过课程管理、账号与学习关系或学生详情相关流程维护。

年级使用固定选项：`初一`、`初二`、`初三`。系统会按工作区记录最近一次升年级年份；每年 9 月 1 日后访问学生列表、学生详情或学生接口时，会自动执行一次升年级：初一升初二，初二升初三，初三标记为“当年初中已毕业”（例如 `2026初中已毕业`）。已毕业年级会保留在学生详情中，可继续查看和编辑其他信息。

#### 查看和编辑学生详情

进入学生详情后，可以查看：

- 所属课程
- 剩余课时/总课时
- 平均得分率
- 出勤率
- 基本信息
- 最近沟通记录
- 课时历史
- 历史上课内容回顾

基本信息可以编辑，包括姓名、年级、联系方式、入学日期、上课频次、学费、总课时、剩余课时、备注和课程。

课时历史会记录手动调整和出勤扣课时等变化。当前规则是：标记“出勤”会扣 1 个剩余课时；同一条考勤重复保存为出勤不会重复扣课时；“补课”暂不扣课时。

历史上课内容回顾来自“排课考勤”的出勤/补课反馈，展示上课内容标签、反馈标签和薄弱点标签。

### 3.3 课程管理

入口：左侧导航“课程管理”

课程管理首页以卡片形式展示课程。卡片显示：

- 课程名称和说明
- 固定课程/定制课程
- 固定上课时间
- 知识点数量
- 选课学生数量

#### 新增课程

点击“新增课程”，填写：

- 课程名称
- 描述
- 类型：固定课程或定制课程
- 默认人数
- 选择学生
- 课程时间

定制课程默认 1 人，只能选择一个学生。固定课程可以选择多个学生，并可设置多个固定时间。

每组课程时间可选择周一到周日中的多个上课日，并提供“工作日”快捷选择。保存时系统会按所选星期自动生成对应的固定排课。

课程时间支持填写开始日期和截止日期：

- 两个日期都不填：按长期固定课处理。
- 只填开始日期：从该日期起展示和生效。
- 只填截止日期：到该日期为止展示和生效。
- 两个日期都填：只在开始日期和截止日期之间展示和生效。

编辑课程时可以修改上课日、开始时间、结束时间、开始日期和截止日期；保存后会同步更新该课程对应的固定排课，避免一次性生成过大的或不合理的日期范围。

修改课程时间时，系统按考勤状态处理旧排课：未记录过考勤的旧课程安排会删除，并按新的课程时间重新生成；已记录考勤的旧课程安排不会被删除，会作为历史排课保留，只在对应考勤日期继续显示，避免家长端课程安排丢失历史课堂或混入无效旧时间。

#### 知识点大纲

进入课程详情后，可以维护知识点大纲：

- 给已有知识点新增子知识点
- 重命名知识点
- 删除知识点
- 批量导入树状知识点

批量导入示例：

```text
有理数
  正负数与数轴
  绝对值与相反数
- 整式的加减
  - 单项式与多项式
```

### 3.4 成绩管理

入口：左侧导航“成绩管理”

成绩管理首页展示所有学生的成绩卡片，包括正式考试、小测平均分和最近趋势。没有成绩的新学生也会出现在列表中。

#### 学生成绩详情

点击学生卡片可查看该学生的成绩明细和曲线。

老师在学生成绩详情中录入新成绩时，点击“保存成绩”会先弹出薄弱点选择框。该弹窗和家长成绩审核通过时保持一致，可以搜索已有薄弱点标签，也可以新增标签；确认后系统保存成绩，并按所选薄弱点创建薄弱点和复习计划。

如果同一个学生、同一学习关系下已经存在同名薄弱点：

- 仍是当前薄弱，或正处在巩固复习中：复用已有薄弱点。
- 已经有待复习计划：不重复生成待复习计划。
- 已经全部完成：不重新激活旧记录，创建一条新的薄弱点并从第 1 次复习开始。

历史成绩支持编辑：

- 考试名称
- 考试类型
- 得分和满分

成绩类型包括：

- 摸底
- 阶段测试
- 随堂小测

#### 家长提交成绩审核

家长端提交的成绩会进入“待审核成绩”区域。

老师或管理员可以：

- 点击“通过”：成绩进入正式成绩记录，家长端曲线会显示。
- 点击“驳回”：填写或使用驳回原因，家长端会看到驳回状态。
- 审核通过时可选择薄弱点标签，系统会为学生创建对应薄弱点和复习计划。

顶部“新建考试”批量录入成绩时，也会在保存前弹出同样的薄弱点选择框；选中的薄弱点会应用到本次所有已填写分数的学生。

### 3.5 学习进度

入口：左侧导航“学习进度”

学习进度首页按学生展示：

- 知识点掌握数量和比例
- 学习中数量
- 当前薄弱点数量

进入学生学习进度详情后，有两个主要入口。

#### 知识点进度

知识点状态有三种：

- 未开始
- 学习中
- 已掌握

点击状态按钮会按“未开始 → 学习中 → 已掌握 → 未开始”循环切换。

#### 薄弱点复习

可以手动添加薄弱点，也可以从已有薄弱点标签中选择。薄弱点创建后会自动生成复习计划。

筛选项包括：

- 全部
- 待复习
- 当前薄弱
- 巩固中
- 已完成

每条薄弱点会显示：

- 描述
- 当前状态
- 创建时间
- 掌握时间
- 已复习次数
- 最近复习时间
- 下次复习阶段和日期
- 是否逾期

可用操作：

- “记住了”：完成当前复习阶段，并进入下一阶段。
- “忘了”：当前薄弱点继续加强复习。
- “已掌握”：从当前薄弱进入巩固流程，后续仍会保留复习安排。

### 3.6 排课考勤

入口：左侧导航“排课考勤”

排课考勤支持月视图和周视图。右侧会显示当前选中日期的课程安排。

#### 新增排课

点击“新增排课”或日期详情中的“+”，选择：

- 学生
- 类型：固定排课或灵活排课
- 固定排课：选择星期
- 灵活排课：选择具体日期
- 开始时间、结束时间
- 备注

固定排课会每周显示；灵活排课只显示在指定日期。

#### 编辑和删除排课

非课程固定排课可直接编辑。删除排课前系统会二次确认。

课程固定排课通常来自课程时间和选课学生关系，不建议在日历里当成单个学生排课随意修改。

#### 记录考勤

每条排课可标记：

- 出勤
- 请假
- 补课

标记“请假”会快速保存；标记“出勤”或“补课”会打开课堂反馈弹窗。

课堂反馈弹窗中可以选择或维护：

- 上课内容标签
- 上课反馈标签
- 薄弱点标签
- 课堂视频

保存后：

- 考勤记录会写入当前日期。
- 上课内容、反馈和薄弱点会进入学生详情的历史回顾。
- 选中的薄弱点会创建为学生薄弱点，并生成复习计划。
- 出勤会扣除 1 个剩余课时。
- 如果选择了课堂视频，视频会关联到本次出勤或补课记录。后续重新选择视频会替换旧视频，也可以在弹窗中删除已上传视频。
- 已经记录过考勤后，排课卡片只显示“已记录：出勤/请假/补课”和“修改”按钮；需要改状态或替换视频时，先点击“修改”，再重新选择考勤状态。

标签支持新增、重命名和删除。删除标签不会清空历史考勤里已经保存的文字标签。

### 3.7 资料中心

入口：左侧导航“资料中心”

资料中心用于上传、搜索、预览、下载和授权资料。

支持上传的文件类型包括：

- PDF
- Word 文档
- HTML/HTM 动画或资料

资料属性包括：

- 标题
- 资料类型：试卷、动画、资料
- 年级
- 科目
- 关键词
- 说明
- 文件

管理员上传时可以选择工作区；老师上传时默认进入自己的工作区。

#### 搜索资料

可按以下条件筛选：

- 标题、说明、年级、关键词
- 年级
- 科目
- 资料属性

#### 授权课程

老师或管理员可以在资料卡片下方勾选“授权课程”，并保存授权。

家长端资料中心只显示已授权给其学习关系相关课程的资料。资料是否可预览/下载由当前账号权限和课程授权共同决定。

### 3.8 作业批改

入口：左侧导航“作业批改”，课程详情页也会显示该课程作业入口。

作业批改流程如下：

- 老师在课程中创建作业，填写标题、说明和可选截止时间。
- 老师上传题目文件和答案/解析文件。文件支持 Word 和 PDF。
- 第一版不再自动识别题目结构。老师在题目结构页填写“填空题有几题、选择题有几题、大题有几题、计算题有几题”，系统按数量生成题号和题型。
- 生成后的题目结构不会直接发布，老师可以继续修改题号、题型、分值、标准答案和解析后确认。
- 发布后，系统按该课程当前有效学生生成作业提交记录。
- 家长端只上传整份答案图片或 PDF，不逐题在线填写。
- 老师按学生提交版本逐题批改，填写每题得分、每题评语、总分和总评。
- 老师批改页使用三栏工作台：左侧预览学生答案，中间预览老师上传的答案/解析，右侧逐题批改；三栏可分别滚动，右侧底部固定保存按钮。
- 老师上传的题目或答案/解析如果是 Word，系统会优先用 LibreOffice 转成 PDF 用于预览；中间栏按钮为“下载答案”，用于下载原始答案文件。
- 如果服务器尚未安装 LibreOffice，Word 预览会退回文本预览；PDF 和图片可以直接预览。
- 老师可在作业详情页删除作业。删除作业会同时删除题目结构、学生提交记录、提交版本和批改记录；上传过的原始文件暂不物理删除，便于后续排查。
- 当前已提供独立的“通用作业答题卡”PDF 模板，暂不在网站内新增下载入口。后续如接入系统生成，可按老师设置的填空题、选择题、大题、计算题数量自动生成对应题号和答题区域。
- 家长端可以查看总分、总评、每题得分和每题评语，也可以在作业详情页下载老师上传的答案/解析文件。
- 截止时间可选；未设置截止时间时不显示逾期，设置后超过截止时间且未提交时显示“已逾期”。
- 已批改后家长仍可重新上传，系统创建新的提交版本；旧提交和旧批改记录保留。

作业状态：

- 作业主状态：草稿、已发布。
- 学生提交状态：待提交、已提交、已批改。
- 逾期为页面展示状态，不单独改变底层提交状态，仍允许补交。

### 3.9 账号与学习关系

入口：左侧导航“账号与学习关系”

仅管理员可见。

#### 管理账号

管理员可以新增或编辑：

- 管理员账号
- 老师账号
- 家长账号
- 演示账号

账号信息包括：

- 姓名
- 登录账号
- 邮箱
- 角色
- 教学科目
- 工作区
- 密码或重置密码
- 家长可见学生

编辑账号时，密码留空表示不修改密码。

编辑账号时可以删除账号。当前正在登录的账号不能删除；如果账号已经关联资料、视频、作业等历史业务记录，系统会阻止删除，避免破坏历史数据。删除家长或老师账号时，其家长可见学生关系和学习关系会随账号清理。

#### 学习关系

学习关系用于定义家长端和小程序中“哪个家长看到哪个孩子、哪个老师、哪个科目/课程”。

学习关系不是登录账号本身。创建家长账号时，先在账号信息里勾选“可见学生”，让家长能看到对应孩子；再在学习关系中把这个家长、孩子、老师和课程/科目绑定起来。学习关系用于成绩提交、课堂记录、学习进度、资料权限、小程序日历等按“家长-学生-老师-科目/课程”精确隔离的数据。

一条学习关系包含：

- 工作区
- 家长
- 学生
- 老师
- 课程，可不绑定
- 科目
- 是否启用

家长端成绩提交、时间管理、学习进度和资料可见范围都依赖学习关系。新增家长账号后，建议立即检查学习关系是否已启用。

学习关系下拉框会联动过滤：

- 选择家长后，学生下拉框只显示该家长账号已勾选的可见学生。
- 选择学生后，课程下拉框只显示该学生当前关联的课程。
- 如果学生或课程下拉框为空，先回到账号编辑或课程管理补齐家长可见学生、课程选课学生。

当前页面展示每个学生的汇总报告卡片，包括：

- 考试次数
- 已掌握知识点比例

目前该页面主要是汇总展示，还不是文件导出按钮。

沟通记录首页按学生展示最近沟通内容。进入学生沟通详情后，可以维护与家长沟通的历史记录。

沟通方式包括：

- 电话
- 微信

学生详情页也会展示最近沟通记录，并支持编辑。

### 3.12 系统设置

### 4.6 作业

## 6. 数据和权限关键规则

### 6.1 出勤与课时

- 出勤扣 1 个剩余课时。
- 同一条考勤重复保存为出勤不会重复扣。
- 补课暂不扣课时。
- 课时变化会写入学生详情的“课时历史”。

### 6.2 薄弱点来源

薄弱点可能来自：

- 学习进度页手动添加
- 排课考勤的课堂反馈
- 老师审核家长提交成绩时选择

创建薄弱点后会自动生成复习计划。

### 6.3 家长可见范围

家长可见数据主要由以下配置决定：

- 家长账号所在工作区
- 家长账号可见学生
- 学习关系是否存在且启用
- 资料是否授权给相关课程

如果家长看不到孩子、成绩、课程或资料，优先检查“账号与学习关系”。

### 6.4 资料权限

资料上传后不等于所有家长可见。需要在资料卡片中保存课程授权。家长端只显示自己有权限访问的资料。

### 6.5 作业批改权限

作业批改权限规则：

- 管理员、老师和演示账号可以在老师端创建、发布和批改作业。
- 作业按工作区隔离，真实工作区和演示工作区不混用。
- 家长只能看到自己绑定学生所属课程的已发布作业。
- 家长只能上传自己孩子的作业答案，只能查看自己孩子的提交版本和批改结果。
- 家长不能查看老师上传的答案/解析文件。
- 作业发布时按课程当前有效学生生成提交记录；课程后续新增学生时，第一版不自动补发历史作业。

### 6.6 演示数据

演示账号用于展示功能，不应混用真实工作区数据。正式业务数据不要执行会重建演示数据或清空数据库的操作。

## 8. 服务器部署与更新

当前线上部署信息：

- 服务器：腾讯云轻量应用服务器
- 系统：Ubuntu
- 主域名：`taotaomath.top`
- www 域名：`www.taotaomath.top`
- 应用端口：`3001`
- Web 入口：Nginx 80/443 反向代理到 `127.0.0.1:3001`
- 进程管理：PM2
- 数据库：SQLite
- 部署目录：`/opt/student-management`
- 当前运行目录：`/opt/student-management/current`
- 数据与上传文件目录：`/opt/student-management/shared`

### 8.1 日常更新

服务器上执行：

```bash
cd /opt/student-management/current
sudo REPO_URL=https://github.com/Haoder413/StuManage.git BRANCH=main bash deploy/deploy-update.sh
```

更新脚本会自动：

- 备份当前 SQLite 数据库
- 拉取最新代码
- 安装依赖
- 生成 Prisma Client
- 同步数据库结构
- 构建项目
- 切换当前版本
- 重启 PM2

### 8.2 回滚

如果更新后出现问题，执行：

```bash
bash /opt/student-management/current/deploy/rollback.sh
```

如果这次更新同时改变了数据库结构，必要时还需要从备份目录恢复对应数据库。

### 8.3 常用检查

检查 PM2：

```bash
sudo pm2 list
```

检查 Nginx：

```bash
sudo nginx -t
sudo systemctl status nginx
```

检查应用端口：

```bash
sudo ss -lntp | grep 3001
```

检查线上域名：

```bash
curl -I https://taotaomath.top
curl -I https://www.taotaomath.top
```

未登录访问后台时返回公开首页是正常现象；登录需要直接访问隐藏登录地址 `/teacher-login-2026`。

### 8.4 SSL 证书

证书由 Let's Encrypt / Certbot 管理。当前记录中的证书到期日为 2026-09-18。

检查自动续期：

```bash
sudo certbot renew --dry-run
```

## 9. 后续更新本手册时的检查清单

硬性规则：任何对话只要新增、修改、确认或废弃了业务信息，都必须同步写入项目文档，不能只留在聊天记录里。当前默认更新本文；如果是部署运维信息，同时更新 `docs/deployment-status.md`；如果是跨轮背景总结，同时更新 `docs/conversation-summary.md`。项目文档只允许使用 Markdown 格式，即 `.md` 文件；不创建、不维护、不要求同步任何 Word 文档格式的项目文档。

以后代码更新后，建议按下面顺序同步检查并更新本文：

1. 查看 `README.md`、`docs/conversation-summary.md`、`docs/deployment-status.md` 是否有新增说明。
2. 检查左侧导航：`src/components/sidebar.tsx`、`src/components/parent-sidebar.tsx`。
3. 检查数据模型：`prisma/schema.prisma`。
4. 检查教师端页面：`src/app/students`、`src/app/courses`、`src/app/exams`、`src/app/progress`、`src/app/schedule`、`src/app/resources`、`src/app/accounts`、`src/app/settings`。
5. 检查家长端页面：`src/app/parent`。
6. 检查小程序页面：`miniprogram/app.json` 和 `miniprogram/pages`。
7. 检查部署脚本和部署说明：`deploy/README.md`、`docs/deployment-status.md`。
8. 如果新增了业务规则，补到“数据和权限关键规则”。
9. 如果新增或改名了入口，补到对应角色章节。
10. 更新本文顶部“更新时间”。

常见需要同步手册的代码变化包括：

- 新增导航菜单或页面
- 改动账号角色权限
- 改动家长可见范围或学习关系
- 改动出勤扣课时、薄弱点复习、成绩审核等业务规则
- 改动资料上传格式或授权方式
- 改动部署目录、域名、端口、证书或更新命令
