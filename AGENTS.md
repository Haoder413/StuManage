# 项目硬性规则

## 业务信息必须同步文档

任何对话只要新增、修改、确认或废弃了业务信息，都必须在同一轮工作中同步写入项目文档，不允许只停留在聊天记录中。

业务信息包括但不限于：

- 功能入口、菜单、页面名称或使用流程
- 角色权限、账号规则、工作区隔离规则
- 学生、课程、排课、考勤、课时、成绩、学习进度、薄弱点、资料、家长端、小程序等业务规则
- 部署域名、端口、服务器路径、备份、证书、更新或回滚流程
- 用户明确要求后续持续记住的项目约定

默认更新位置：

- 用户操作和业务规则：`docs/操作手册.md`
- 部署和运维信息：`docs/deployment-status.md`
- 会话阶段性总结或跨轮上下文：`docs/conversation-summary.md`

项目文档只允许使用 Markdown 格式，即 `.md` 文件；不创建、不维护、不要求同步任何 Word 文档格式的项目文档。
